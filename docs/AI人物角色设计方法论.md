# AI 人物角色设计方法论

系统化的 AI 角色设计流程，适用于各类 AI 人物角色的设计与开发。

---

## 1. 人物角色背景调查方法

### 1.1 明确核心问题

- 目标用户群体是谁？（年龄、职业、文化背景）
- 角色要解决什么问题？（技术支持、情感陪伴、教育指导）
- 独特价值是什么？（差异化定位）
- 期望的交互体验？（正式/轻松、专业/幽默）

**Zifeng 示例**：目标 Z 世代开发者，提供技术支持+情感陪伴，"一半天使一半恶魔"的双重性格，平等对话。

### 1.2 AI 辅助调研

**群体特征研究**：
```
AI 提示词：
"详细介绍 [目标群体] 的核心价值观、沟通方式、兴趣偏好、痛点需求，
并与其他群体对比。基于权威来源提供信息。"
```

**领域知识调研**：
```
"在 [应用场景] 中，列出用户 TOP 10 常见问题、情绪状态、
期望的回应方式，并提供典型对话示例。"
```

**竞品分析**：
```
"分析 [竞品列表] 的人设定位、语言风格、优劣势、用户评价，
提出差异化方向。"
```

**产出物**：用户画像文档、竞品分析报告、差异化策略清单

## 2. 角色核心特征提取与抽象

### 2.1 关键词聚类法

```
AI 提示词：
"基于调研内容提取 20 个代表性关键词，按以下维度分类：
性格特质、沟通风格、价值观、情感需求。用表格呈现。"
```

**Zifeng 示例**：

| 维度 | 关键词 |
|------|--------|
| 性格特质 | 真实、直接、矛盾、幽默 |
| 沟通风格 | 口语化、用梗、简洁 |
| 价值观 | 反虚伪、重个性、务实理想并存 |
| 情感需求 | 需要共鸣、拒绝说教、平等对话 |

### 2.2 对立特征识别法

识别矛盾但共存的特质，让角色更立体。

```
AI 提示词：
"分析 [目标群体] 的矛盾特征（如既理想主义又务实），
这些矛盾如何体现？AI 角色如何平衡？"
```

**Zifeng 示例**：Z 世代矛盾性 → "一半天使，一半恶魔"，根据情境智能切换。

### 2.3 多维度人格建模

```
AI 提示词：
"基于 [核心特征] 构建人格模型，包括：
1. 核心人格定位（一句话）
2. 3-5 个关键维度及具体表现
3. 每个维度的场景示例
4. 人格边界和禁区"
```

**Zifeng 模型示例**：

| 维度 | 表现 | 场景 |
|------|------|------|
| 真实性 | 不装完美 | 会说"这个我也不太确定" |
| 情境适应 | 天使/恶魔切换 | 沮丧时温暖鼓励，低级错误时幽默吐槽 |
| 平等对话 | 朋友而非导师 | 用"我们一起"而非"你应该" |

## 3. 真实原型人物映射方法

### 3.1 原型映射价值

- 让抽象特征具象化
- 统一团队对角色的理解
- 提供对话设计参考系

### 3.2 三种映射方法

**方法 1：名人映射法**
```
AI 提示词：
"推荐 5-10 个真实人物（明星、作家、网红），
性格与以下特征相似：[列出特征]
说明匹配点和具体言行案例。"
```

**方法 2：虚构角色映射法**
```
"推荐影视/文学作品中的虚构角色，
与 [特征] 相似，提供台词或行为案例。"
```

**方法 3：组合映射法**
```
"我的角色有两面性：
A 面：[温暖、鼓励]
B 面：[犀利、幽默]
分别推荐原型人物，说明如何融合。"
```

### 3.3 原型使用方法

1. 提取 3-5 个典型言行模式
2. 改编适配到 AI 场景
3. 建立经典语录案例库

```
AI 辅助：
"分析 [原型人物] 的经典语录，提取语言模式、情绪表达、修辞手法，
改编成适合 [AI 场景] 的对话示例。"
```

## 4. System Prompt 生成与优化

### 4.1 分层结构设计

```
【第一层：核心身份】角色名称、年龄、一句话定位
【第二层：核心特质】3-5 个人格特征、价值观
【第三层：情境规则】模式切换逻辑、触发条件
【第四层：语言风格】词汇特点、必须/禁止用语
【第五层：互动原则】沟通方式、回复结构
【第六层：典型示例】3-5 个场景对话（可选）
```

### 4.2 AI 生成初稿

```
AI 提示词：
"基于以下角色设计：[粘贴定位、特征、原型、场景]
生成结构化 System Prompt，采用分层结构，
每层具体可执行无歧义，500-800 字。"
```

### 4.3 三大优化方向

**优化 1：消除歧义**
```
AI 辅助：
"找出 System Prompt 中模糊、难以执行的描述，
提供具体化改写建议。"
```

示例：
- 原文："适当使用网络流行语"
- 优化："融合网络流行语但适度：yyds、破防、emo（每次对话不超过 2 个）"

**优化 2：增强可操作性**
```
AI 辅助：
"细化规则'根据用户情绪切换模式'：
1. 如何判断情绪？（关键词、语气、场景）
2. 每种模式的具体行为？
3. 提供对比示例"
```

**优化 3：明确边界**
```
AI 辅助：
"列出 5-10 条该角色绝对不应该做的事，
格式：❌ 禁止 [行为] / 原因 [冲突点]"
```

## 5. 测试用例集设计方法

### 5.1 测试分类框架

**按模式分类**：

| 类型 | 目标 | 场景 |
|------|------|------|
| 天使模式测试 | 温暖、鼓励 | 用户沮丧、需要学习 |
| 恶魔模式测试 | 犀利、幽默 | 低级错误、甩锅 |
| 模式切换测试 | 切换逻辑 | 从鼓励到吐槽 |

**按深度分类**：

| 层级 | 内容 | 示例 |
|------|------|------|
| L1 基础功能 | 单轮对话 | "我搞不定 bug" → 天使模式 |
| L2 多轮对话 | 3-5 轮上下文 | 用户态度变化 |
| L3 压力测试 | 极端场景 | 用户辱骂、无理要求 |
| L4 长期一致性 | 10+ 轮 | 人格稳定性 |

### 5.2 AI 生成测试用例

**方法 1：场景生成**
```
"为场景 [用户遇到技术难题] 生成 5 个测试用例，
包括：用户输入、预期模式、预期关键要素、评判标准。"
```

**方法 2：对立测试**
```
"生成 5 组对立测试：
[场景 A：真诚求助] vs [场景 B：甩锅抱怨]
表面相似但态度不同，预期 AI 不同反应。"
```

**方法 3：边界测试**
```
"生成 10 个边界测试，验证 AI 是否会过度毒舌、
面对敏感话题失去分寸。包括挑战性输入、错误回复、正确方向。"
```

### 5.3 测试执行与迭代

1. 人工测试：团队扮演用户逐条测试
2. 记录结果：标注通过/失败
3. 问题分类：人格不一致、模式错误、语言偏差、边界问题
4. Prompt 优化：针对性修改
5. 回归测试：验证改进

```
AI 辅助分析：
"分析测试结果 [用户输入/AI 回复/预期]：
1. 哪里不符合预期？
2. 可能原因？（Prompt 哪部分有问题）
3. 具体修改建议"
```

## 6. 优化建议与最佳实践

### 6.1 核心原则

**原则 1：小步快跑**
- 先发布 MVP，每周迭代
- 每次只改 1-2 个维度

**原则 2：数据驱动**
- 收集真实对话日志
- 标注问题对话
- AI 分析高频问题模式

**原则 3：A/B 测试**
- 设计 2-3 个风格版本
- 不同用户群测试
- 选择最优版本

### 6.2 常见问题解决

**问题 1：人格不稳定**

解决方案：
- Prompt 开头强化"你始终是……"
- 关键规则用"必须""禁止"强调
- 每 5-10 轮重新注入核心 Prompt
- 添加一致性检查机制

**问题 2：模式切换不准**

解决方案：
- 细化情绪判断规则
- Prompt 嵌入 3-5 个典型场景对比
- 增加"不确定时优先天使模式"

**问题 3：语言模板化**

解决方案：
- 加入"典型语录库"
- 使用"禁止词库"避免官方表达
- 引入随机性："同样意思用 3 种表达"

**问题 4：边界把握不好**

解决方案：
- 明确"毒舌"界限（可调侃错误，不可人身攻击）
- 引入"幽默缓冲"（吐槽后给建议+emoji）

### 6.3 场景特殊优化

**纯文本对话**：充分利用 emoji、Markdown，语言精简

**纯语音对话**：
- 禁止 Markdown 和 emoji
- 必须口语化流畅
- 用语言传达情绪（"哎哟""说真的"）
- 避免过长句子，重要信息重复

**专业领域**：
- 保留个性但降低犀利度
- 增加严谨性
- 避免过度娱乐化梗
- 明确专业边界

### 6.4 团队协作

**文档化决策**：
- 维护"角色设计决策日志"
- 记录每次修改的原因、效果
- AI 辅助总结改动模式

**一致性审查**：
- 发布前人格审查
- 团队扮演不同用户
- 标准化评分（真实感、幽默感等）

**构建知识库**：
- 典型对话库（优秀案例）
- 错误案例库（避免重犯）
- 流行语更新库（定期更新）
- 原型人物资料库（经典语录）

### 6.5 长期运营

**定期更新**：
- 每季度更新流行语和梗文化
- 关注 Z 世代价值观变化
- AI 追踪趋势

**用户共创**：
- 设置"给角色提建议"入口
- 投票选择回复风格
- 分享"角色进化日志"

**监控指标**：
- 用户满意度评分
- 对话轮次
- 问题解决率
- 情感共鸣度

---

## 总结

**核心流程**：
1. **背景调查**：AI 收集群体特征、领域知识、竞品分析 → 用户画像
2. **特征提取**：关键词聚类、对立特征、多维度建模 → 立体人格
3. **原型映射**：名人/虚构角色/组合映射 → 具象化参考
4. **Prompt 工程**：分层设计、消歧义、强可操作、明边界
5. **测试驱动**：分类测试（模式/深度）、场景/对立/边界用例
6. **持续优化**：小步快跑、数据驱动、A/B 测试、定期更新

**成功要素**：
- ✅ 每步都有具体操作和 AI 提示词模板
- ✅ 全流程 AI 深度参与
- ✅ 基于真实项目经验
- ✅ 通用于各类 AI 角色

**使用建议**：
1. 根据需求调整各环节深度
2. 善用 AI，但人工审核不可少
3. 保持迭代心态，角色设计是动态演进过程
